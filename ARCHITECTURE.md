## Stack
- Next.js 14+ App Router, TypeScript, Tailwind CSS
- Supabase (PostgreSQL + Auth + RLS)
- ElevenLabs Conversational AI (voice lessons)
- Anthropic Claude API (exercise generation, conversation analysis)
- Vercel (hosting, auto-deploy on git push to main)
- Resend (transactional emails from noreply@godoj.co)

## Auth Flow
- Magic links via Supabase Auth (no passwords)
- Middleware: src/middleware.ts — protects all /app routes
- API routes check session via Supabase — 401 is CORRECT for unauthenticated requests
- Admin check: user role in users table must be "admin"

## i18n System
- Core: src/lib/i18n.ts — translation dictionaries
- Wrapper: src/components/locale-wrapper.tsx — provides translations to components
- User's native_language stored in user_profiles table
- Language selection happens in onboarding step 1
- After selection, ALL UI should render in that language

## Database (Supabase) — Key Tables
- users — extends auth.users (display_name, role, native_language, onboarding_complete)
- user_profiles — learning preferences (target_language, level, selected tutor/agent)
- streaks — daily streak tracking (current_streak, longest_streak, weekly_minutes)
- achievements — badge definitions
- user_achievements — earned badges per user
- vocabulary — saved words from lessons
- lessons — lesson history (duration, topic, summary)
- agents_config — AI tutor configurations (ElevenLabs agent IDs)

## Voice Lessons Flow
1. User clicks "Start conversation" → POST /api/lessons/start (creates lesson record)
2. Frontend connects to ElevenLabs Conversational AI agent (WebSocket)
3. Real-time voice conversation happens client-side
4. On end → POST /api/lessons/end (saves summary, updates streaks)

## Exercise Generation
- POST /api/exercises/generate — requires auth session
- Uses Claude API (Haiku) to generate exercises
- Must generate in user's target_language (NOT always English)
- 7 exercise types: multiple choice, fill blank, translation, etc.

## Common Gotchas
- API routes returning 401 = CORRECT behavior for unauthenticated requests, NOT a bug
- Onboarding must be completed before accessing dashboard
- ElevenLabs connection requires valid agent ID from agents_config table
- Vocabulary entries must have correct language field matching user's target_language
- After git push to main, Vercel auto-deploys — wait ~90 seconds before testing
